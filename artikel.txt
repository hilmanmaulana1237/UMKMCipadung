Development of an AI Video Content Generator with Rule-Based Prompt Generation for UMKM Digital Advertising
Hilman Maulana¹, et al.

¹ Informatics Engineering, Bandung, Indonesia

Abstract
Micro, Small, and Medium Enterprises (MSMEs/UMKM) in developing countries face significant challenges in creating professional digital advertising content due to limited technical expertise and financial resources. Most existing Generative AI tools require users to manually craft effective prompts—a skill that is inaccessible to the majority of UMKM operators. This paper presents the development of MUDAPRENEUR.AI, a Progressive Web Application that integrates a Rule-Based Prompt Generation method to automatically construct structured prompts for AI video content generation, thereby eliminating the need for users to understand prompt engineering. The system is built using Laravel 12 (PHP) for the backend, React with TypeScript via Inertia.js for the frontend, and integrates with external Generative AI APIs (OpenRouter for text generation and Kie AI/Veo 3.1 for video generation). Users simply input their business data—product name, store category, and reference photo—and the backend automatically generates a comprehensive, structured video prompt using predefined templates and rules. The prompt is then dispatched to a Generative AI API to produce a User-Generated Content (UGC)-style advertising video. Functional testing demonstrates that all core AI content generation workflows operate correctly, including prompt construction, API integration, asynchronous task management, and quota enforcement. The primary contribution of this work is a practical framework that democratizes AI-powered video advertising for technically non-expert UMKM operators through backend prompt automation.

Keywords: Generative AI, Rule-Based Prompt Generation, AI Content Generator, UMKM, Digital Advertising, Web Application, Progressive Web Application

1. Introduction
Micro, Small, and Medium Enterprises (MSMEs), known as UMKM (Usaha Mikro, Kecil, dan Menengah) in Indonesia, constitute over 99% of the nation's business entities and contribute approximately 61% of the Gross Domestic Product [1]. Despite their economic significance, UMKM operators—particularly in rural and semi-urban areas such as Kelurahan Cipadung in Bandung—face persistent challenges in digital marketing and online promotion [2]. Professional advertising content creation, especially video advertising, typically requires specialized skills in videography, copywriting, graphic design, and increasingly, prompt engineering for AI-assisted tools [3].

The rapid advancement of Generative AI technologies, including large language models (LLMs) and text-to-video diffusion models, has opened new possibilities for automated content creation [4]. Tools such as OpenAI's GPT series, Google's Gemini, and video generation models like Sora and Veo have demonstrated remarkable capabilities in generating text, images, and video from natural language prompts [5]. However, the effective use of these tools demands proficiency in prompt engineering—the craft of formulating precise, contextually rich instructions to guide AI model outputs [6]. This requirement creates a significant adoption barrier for UMKM operators who lack technical literacy.

Several existing platforms provide AI-powered content generation, but most require users to manually write or customize prompts [7]. This paradigm assumes a level of technical understanding that is misaligned with the skill profile of the average UMKM operator in developing regions. The research gap addressed in this work is the absence of systems that fully automate the prompt construction process, enabling non-technical users to generate professional-grade advertising content by simply providing basic business information.

This paper presents the development of MUDAPRENEUR.AI, a web application that implements a Rule-Based Prompt Generation method to bridge the gap between Generative AI capabilities and UMKM user accessibility. The system automatically transforms structured business data (product name, store category, description, location, and reference photographs) into comprehensive, multi-section prompts that are optimized for AI video generation APIs. The key contributions of this work are:

The design and implementation of a Rule-Based Prompt Generation engine that eliminates user-facing prompt engineering.
The integration of a full-stack web application with multiple Generative AI APIs for text, image, and video content generation.
A complete AI Content Generator ecosystem for UMKM digital advertising, including video generation, poster creation, copywriting automation, and landing page building.
A practical deployment targeting rural UMKM communities in Cipadung, Bandung, demonstrating the feasibility of AI-assisted digital marketing for non-technical users.
2. Method
2.1 System Overview
The development of MUDAPRENEUR.AI follows a software development / system implementation research methodology. The system is designed as a Progressive Web Application (PWA) with a mobile-first, native-like user experience. The development process encompasses requirement analysis, system architecture design, implementation, and functional testing.

The application serves as a comprehensive digital ecosystem for UMKM in Kelurahan Cipadung, Bandung. While the platform includes marketplace, order management, courier logistics, and affiliate marketing features, the primary research focus of this paper is the AI Content Generator module, specifically the AI Video Content Generator with backend-controlled prompt automation.

Based on field observation and analysis of UMKM needs in Cipadung, the following key requirements were identified:

No.	Requirement	Description

R3	Backend prompt automation	All prompt construction logic must reside in the backend, invisible to the user
R4	Multiple content types	The system should support video scripts, promotional videos, posters, copywriting, and landing pages
R5	Quota management	Each UMKM must have a defined quota for AI-generated content to manage API costs
R6	Asynchronous processing	Video generation (which takes minutes) must be handled asynchronously with status polling
R7	Mobile-first interface	The user interface must be optimized for mobile devices commonly used by UMKM operators

2.4 Rule-Based Prompt Generation
The central algorithmic contribution of this work is the Rule-Based Prompt Generation method. Unlike ad-hoc prompt engineering, which requires human expertise to manually compose prompts, Rule-Based Prompt Generation is a computational procedure that systematically constructs structured prompts from predefined templates and user-supplied data according to deterministic rules.

Clarification on terminology: Prompt engineering refers to the design-time technique used by developers to craft effective prompt templates. The runtime computational procedure implemented in the system is Rule-Based Prompt Generation, where user inputs are programmatically transformed into structured prompts using predefined backend templates and rules. The generated prompt is then submitted to a Generative AI API to produce advertising content.

The Rule-Based Prompt Generation process is implemented in the AIPromptService class (file: app/Services/AIPromptService.php). The method operates through the following stages:

Table 2: Rule-Based Prompt Generation Stages

Stage	Process	Implementation Detail
1. Data Collection	User inputs basic business data via the frontend form	Product name, reference photo, store category (from UmkmStore model)
2. Template Selection	Backend selects the appropriate prompt template based on content type	constructVideoPrompt() method in AIPromptService
3. Variable Injection	User data is injected into template placeholders	String interpolation with $productName variable
4. Scene Structuring	The template defines a multi-scene video structure	Opening, activity, detail, interaction, closing scenes with timing constraints
5. Visual Constraint Specification	Technical parameters for the AI model are appended	Aspect ratio, duration limits, camera style, visual constraints
6. Negative Prompt Attachment	Undesired output characteristics are explicitly excluded	Rigid poses, static camera, cartoon style, low quality
7. Prompt Dispatch	The complete prompt is sent to the Generative AI API	Via KieAIService::createVideoTask() with the prompt and reference image
The following pseudocode illustrates the core logic:


FUNCTION constructVideoPrompt(productName):
    prompt ← LOAD_TEMPLATE("UGC_VIDEO_TEMPLATE")
    prompt ← INJECT(prompt, "productName", productName)
    prompt ← APPEND(prompt, SCENE_DEFINITIONS)
    prompt ← APPEND(prompt, TIMING_RULES)
    prompt ← APPEND(prompt, VISUAL_CONSTRAINTS)
    prompt ← APPEND(prompt, NEGATIVE_PROMPT)
    RETURN prompt
END FUNCTION
The actual implementation in AIPromptService::constructVideoPrompt() generates a prompt with the following structured sections:

Concept Statement: Defines the video as a UGC recommendation of the product
Scene Details: Specifies subject behavior, product handling, facial expressions, and camera movement
Lighting Directives: Natural, window-lit, non-studio aesthetics
Dialog Script: Pre-written Indonesian-language endorsement (maximum 6 seconds)
Timing Breakdown: Second-by-second scene allocation (0–2s: camera look, 2–6s: dialog, 6–8s: product focus)
Audio Specifications: Natural voice, ambient environmental sounds
Style Definition: UGC, authentic, non-commercial appearance
Critical Visual Constraints: Instructions to preserve product appearance from the reference image

2.7 Functional Testing
Given the nature of this research as a system development study, the evaluation method employed is functional testing (also referred to as black-box testing). Functional testing verifies that each system feature operates according to its specification without examining internal code structures. No user acceptance testing (UAT) with external participants was conducted in this phase; such evaluation is designated as future work.

The testing approach covers:

Feature correctness: Each AI content generation endpoint is tested for correct input validation, prompt construction, API integration, and output delivery.
Error handling: Scenarios including API failures, quota exhaustion, invalid inputs, and timeout conditions are verified.
Asynchronous workflow: The polling mechanism for video/poster generation status is tested for correct state transitions.
Quota enforcement: The system's limits (1 video per UMKM, 2 posters per UMKM) are tested for correct enforcement.

3.2 AI Video Content Generator Workflow
The AI Video Content Generator feature operates through the following user-facing workflow:

User accesses the AI Content page at /umkm/ai-content.
User selects the video generation option and inputs:
Product name (text field, required, max 255 characters)
Reference photo (image upload, required, max 10 MB; or a previously generated UGC photo)
User clicks the generate button.
The system validates input, checks quota (maximum 1 video per UMKM), and initiates the generation process.
The frontend displays a loading state and begins polling the backend every 10 seconds.
Upon completion, the generated video is displayed inline with download and sharing options.
The entire process requires no prompt writing from the user. The system handles all prompt construction, API communication, and result delivery transparently.

3.3 Backend Prompt Automation
The backend prompt automation is the core technical contribution of this work. The AIPromptService::constructVideoPrompt() method (lines 71–119 of AIPromptService.php) generates a comprehensive prompt of approximately 1,500 characters that includes:

Video concept definition: "A UMKM operator / ordinary individual recommending the product ($productName) naturally, like a TikTok or WhatsApp Story video"
Detailed scene instructions: Subject behavior, hand movements, camera movement (handheld, slightly shaky)
Lighting and background specifications: Natural daylight, simple background (home, kitchen, small shop)
Pre-written dialog in Indonesian: "Produknya bagus banget, aku udah pakai sendiri. Kalian wajib coba!" (6-second maximum)
Precise timing breakdown: 0–2s (eye contact + smile), 2–6s (dialog + product showcase), 6–8s (product focus + positive expression)
Critical visual constraints: 7 rules ensuring the AI preserves the exact product appearance from the reference photo
This prompt automation means that a UMKM user who inputs only "Nasi Goreng Spesial" as the product name and uploads a food photo will receive the same quality of structured prompt that a prompt engineering expert would produce manually.

Furthermore, the system includes an optional AI-enhanced prompt mode (enhancePromptWithAI() method), which uses a secondary LLM call to further optimize the prompt for TikTok-style UGC aesthetics while maintaining Indonesian context and product visual fidelity.

3.6 Discussion of Benefits for UMKM
The Rule-Based Prompt Generation approach provides several tangible benefits for UMKM operators:

1. Elimination of Technical Barriers. The system completely removes the need for prompt engineering knowledge. A UMKM operator selling "Nasi Goreng" in Cipadung need only type the product name and upload a photo. The backend automatically generates a 1,500+ character prompt with cinematic scene descriptions, timing constraints, lighting directives, and visual fidelity rules—knowledge that would typically require training in AI prompting techniques.

2. Consistent Output Quality. Because prompts are generated from optimized templates, the quality of generated content is consistent regardless of the user's technical proficiency. The template includes specific instructions for:

Handheld camera movement (authentic UGC feel)
Natural Indonesian atmosphere and lighting
Product visual preservation constraints
Time-limited dialog to prevent voice-over truncation
3. Cost-Effective API Usage. The system implements quota management (1 video and 2 posters per UMKM) and uses the veo3_lite model (the most cost-effective variant), demonstrating awareness of operational sustainability for community-serving applications.

4. Multi-Tier AI Architecture. The ApiSetting model supports three API tiers (primary, secondary, video), allowing administrators to configure different AI models for different task complexities:

Primary: Complex tasks (business mentoring, prompt enhancement) — e.g., GPT-4o-mini
Secondary: Simple tasks (descriptions, captions) — e.g., Gemini Flash, DeepSeek
Video: Video/image generation — Kie AI (Veo 3.1, Seedream 4.5)
5. Graceful Degradation. The system includes comprehensive fallback mechanisms:

Fallback descriptions when AI API fails (category-specific templates in getFallbackDescription())
Fallback captions when copywriting generation fails
Fallback video scripts when JSON parsing fails
Automatic stuck job cleanup (15-minute timeout)
3.7 Limitations
The following limitations are identified based on the current implementation:

No quantitative user evaluation. The system has not been evaluated through formal user acceptance testing (UAT) with UMKM operators. All testing in this study is limited to functional testing.
Output quality dependency on external APIs. The quality of generated videos and posters depends entirely on the capabilities and availability of the external Generative AI APIs (Veo 3.1, Seedream 4.5). The system has no control over AI model improvements or deprecations.
Limited quota allocation. The current quota (1 video, 2 posters per UMKM) is restrictive. A more flexible quota system with periodic resets or tiered plans would improve usability.
Single language support. The current prompt templates are designed for Indonesian-language content. Multilingual support is not implemented.
No A/B testing of prompt templates. The current implementation uses a single prompt template per content type. Comparative evaluation of alternative template designs could potentially improve output quality.
Localhost image handling. The system includes workarounds for localhost development (placeholder images for testing), indicating that local development and production deployment paths differ.
4. Conclusion
This paper has presented the development of MUDAPRENEUR.AI, a Progressive Web Application that implements Rule-Based Prompt Generation for automated AI video content generation targeting UMKM digital advertising needs. The system successfully demonstrates that backend prompt automation can effectively bridge the gap between advanced Generative AI capabilities and the limited technical expertise of UMKM operators.

The key findings and contributions of this work are:

Rule-Based Prompt Generation is a viable method for democratizing Generative AI access for non-technical users. The AIPromptService class demonstrates that deterministic, template-based prompt construction can produce prompts of sufficient quality for Generative AI video models, without requiring user involvement in the prompt engineering process.

The system architecture—combining Laravel 12, React/TypeScript with Inertia.js, and multiple external AI APIs—provides a practical and reproducible framework for integrating Generative AI into web applications serving under-resourced communities.

Functional testing confirms that all core AI content generation features operate correctly, including prompt construction, asynchronous task management, quota enforcement, and graceful error handling.

The AI Content Generator ecosystem extends beyond video generation to include poster creation, copywriting automation, product description generation, visual description generation, and landing page building, providing a comprehensive suite of AI-powered marketing tools for UMKM.

Future work includes:

Conducting user acceptance testing (UAT) with UMKM operators in Cipadung to evaluate usability and perceived usefulness.
Implementing A/B testing of prompt templates to optimize content quality.
Expanding the system to support additional AI models and content types.
Developing a feedback loop where user ratings of generated content inform prompt template improvement.
Conducting comparative studies between manual prompt engineering and rule-based prompt generation output quality.
